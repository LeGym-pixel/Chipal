# SVPN → PeladuVPN

Минималистичный Android VPN-клиент в тёмном кибер-оккультном стиле:
вставить ключ/конфиг/ссылку, выбрать сервер, подключиться под вращение
Уробороса и музыку, сменить язык (RU/KK/EN), настроить параметры
туннелирования.

## Что добавлено под бренд PeladuVPN

- **UI**: чёрный фон, зелёный/белый акцент, едва заметная сетка
  пентаграмм на фоне (`ui/theme/PentagramBackground.kt`).
- **Гексагональная кнопка** подключения в стиле тайлов Civilization V
  (`ui/components/HexConnectButton.kt`), окружённая PNG-Уроборосом.
- **Анимация Уробороса** (`ui/components/OuroborosRing.kt`):
  - в покое — не крутится
  - при нажатии/во время подключения — быстро крутится по часовой
  - когда VPN подключён — крутится медленно, по часовой
  - при отключении — плавно докручивается до исходного положения и
    останавливается
- **Музыка** (`audio/PeladuMusicController.kt`): не играет, пока VPN
  отключён или подключается; запускается ровно в момент успешного
  подключения на скорости 0.7x и за ~1.5 сек разгоняется до 1.0x;
  при отключении продолжает играть на обычной скорости и за ~1.5 сек
  плавно замедляется до полной остановки.
- **Ссылки конфигурации** (`data/PeladuLinkParser.kt`):
  - `peladu://:peladu:<payload>` — префикс буквально заменяется на
    `vpn://`, payload расшифровывается AES-256-GCM ключом, зашитым в
    приложение → помечается как официальный конфиг.
  - `vpn://<payload>` (introduced напрямую, без обёртки) — payload
    просто base64, без шифрования → импортируется, но помечается как
    неофициальный, пользователю показывается предупреждение.
  - Оба формата also перехватываются как deep link (`intent-filter` в
    манифесте на схемы `peladu` и `vpn`) — открытие такой ссылки в
    браузере/мессенджере сразу откроет экран добавления сервера.

**Честно о "шифровании":** ключ AES зашит в APK, так что это не защита
от разбора приложения, а просто способ отличить конфиги, сгенерированные
вашим собственным сервером/генератором ссылок, от произвольных `vpn://`
ссылок. Реальная защита самого VPN-трафика обеспечивается протоколом
WireGuard и к этой схеме отношения не имеет.

**Музыкальный трек**: `app/src/main/res/raw/peladu_theme.mp3` — файл,
который вы прислали. Если планируете публиковать приложение, стоит
убедиться, что у вас есть права на использование этой записи.

**News-кнопка**: открывает `https://suddenly-suitable-citrine.tilda.ws/`
во внешнем браузере через `Intent.ACTION_VIEW` — само приложение
содержимое этой страницы не проверяет и не хранит.

## Что уже сделано

- **Список серверов** — добавление через вставку текста конфига (.conf)
  или импорт файла; хранение локально (Jetpack DataStore).
- **Подключение/отключение** — через официальную библиотеку
  `com.wireguard.android:tunnel` (Apache 2.0), тот же протокол, что
  использует WireGuard/AmneziaWG.
- **Смена языка** — RU / KK / EN, применяется мгновенно через
  AndroidX per-app language API. Все тексты вынесены в `res/values(-ru|-kk|-en)/strings.xml`.
- **Настройки туннелирования на сервер**: MTU, DNS, Allowed IPs,
  Persistent Keepalive, а также поля обфускации AmneziaWG (Jc, Jmin,
  Jmax, S1, S2) — сохраняются в профиле сервера.

## Важное ограничение (обфускация AmneziaWG)

Библиотека `com.wireguard.android:tunnel`, которая реально поднимает
туннель, понимает **только стандартный WireGuard**. Поля Jc/Jmin/Jmax/
S1/S2/H1-H4 сейчас парсятся, хранятся и отображаются в UI, но
**не влияют на сам туннель** — движок их просто игнорирует.

Если нужна настоящая обфускация трафика (это то, что отличает AmneziaWG
от ванильного WireGuard и помогает обходить блокировки), есть два пути:

1. Собрать и подключить `amneziawg-android` как модуль/AAR вместо
   `com.wireguard.android:tunnel` — у него тот же Java/Kotlin API
   (`Backend`, `Tunnel`, `Config`), поэтому `TunnelManager.kt`
   потребует минимальных правок.
   Репозиторий: https://github.com/amnezia-vpn/amneziawg-android
2. Использовать основной `amnezia-client` как есть и просто
   переоформить его UI (что мы обсуждали раньше) — там обфускация
   уже работает "из коробки".

Файл `TunnelManager.kt` написан так, чтобы бэкенд можно было заменить,
не переписывая экраны.

## Как получить готовый APK без своего компьютера (GitHub Actions)

В проекте уже есть `.github/workflows/build.yml`. Если залить проект
в свой репозиторий на GitHub — APK соберётся автоматически:

1. Создайте пустой репозиторий на github.com (публичный или приватный).
2. Залейте туда содержимое этой папки:
   ```
   cd SVPN
   git init
   git add .
   git commit -m "SVPN"
   git branch -M main
   git remote add origin https://github.com/ВАШ_ЛОГИН/ВАШ_РЕПО.git
   git push -u origin main
   ```
3. Откройте вкладку **Actions** в репозитории на GitHub — сборка
   запустится сама (займёт 2-5 минут).
4. Когда сборка позеленеет, зайдите в неё → внизу раздел **Artifacts**
   → скачайте `SVPN-debug-apk` (это zip с .apk внутри).
5. Перекиньте .apk на телефон и установите (может понадобиться
   разрешить "Установку из неизвестных источников").

Это debug-сборка — она не подписана для публикации в Google Play, но
прекрасно ставится и работает на обычном телефоне для тестирования.

## Сборка

1. Установите Android Studio (последняя стабильная версия).
2. Откройте папку `SVPN` как проект ("Open" → выбрать эту папку).
3. Android Studio предложит создать/почистить Gradle wrapper — согласитесь
   (в архиве нет бинарного `gradle-wrapper.jar`, только конфиг; студия
   сгенерирует его сама при первом открытии, либо выполните `gradle wrapper`
   вручную, если у вас установлен Gradle).
4. Дождитесь Gradle sync.
5. Запустите на устройстве/эмуляторе (Run ▶). При первом подключении
   система спросит разрешение на создание VPN-профиля — это стандартное
   системное окно Android, обойти его нельзя (и не нужно).

## Структура

```
app/src/main/java/com/svpn/app/
  MainActivity.kt       — весь UI на Jetpack Compose (4 экрана)
  SvpnApplication.kt     — применяет сохранённый язык при старте
  data/
    ServerProfile.kt     — модель сервера/конфига
    WgConfigParser.kt     — парсер/сериализатор .conf
    SvpnRepository.kt     — локальное хранилище (DataStore + JSON)
  vpn/
    TunnelManager.kt       — обёртка над WireGuard GoBackend
    SvpnTunnelService.kt    — системный VpnService
  util/
    LocaleHelper.kt         — переключение языка
```
